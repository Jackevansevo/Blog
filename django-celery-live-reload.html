Django Celery Live Reload

Published: Mon 19 September 2022
By Jack Evans

In misc.

In part 1 after deining our new square task, we had to restart our celery worker manually. It would be nice if we could configure our celery worker process to autoreload in development the same way that the Django runserver command does when files are changed.

To resolve this, we'll need a management command. Management commands are defined per app so lets go ahead and create one inside posts

Create a new folder under:

mkdir -p posts/management/commands

Then in a new file posts/management/commands/celery.py

import shlex
import subprocess

from django.core.management.base import BaseCommand
from django.utils import autoreload


def restart_celery():
    cmd = "pkill celery"
    subprocess.call(shlex.split(cmd))
    cmd = "celery -A blog worker -l INFO"
    subprocess.call(shlex.split(cmd))


class Command(BaseCommand):
    def handle(self, *args, **options):
        print("Starting celery worker with autoreload...")
        autoreload.run_with_reloader(restart_celery)

Now you should be able to run:

pipenv run ./manage.py celery
